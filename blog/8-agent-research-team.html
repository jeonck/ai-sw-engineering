8명 AI 에이전트팀으로 업무 리서치 10분 안에 끝내기

2026년 5월 8일 · 약 6분
jeonck
SW Engineer

챗봇 한 명에게 다 맡기는 리서치는 왜 실패하는가. 40분짜리 리서치를 10분으로 단축한 8명 에이전트팀 워크플로우 스킬을 Claude Code 스킬로 구현하고, AI 광통신 기술 리서치로 실전 검증했다.

왜 챗봇 한 명에게 다 맡기면 안 되는가​

예전 업무용 심층 리서치 1건에 40분이 걸렸다. 제미나이 심층리서치 10분, 팩트체크·교차검증 30분. 3건 중 1건은 틀린 정보가 섞여 있었다.

문제는 분업의 부재였다. 한 에이전트에게 전부 맡기면 두 가지 병리가 발생한다.

  • 깊이의 희생 — 시장도, 재무도, 기술도 모두 표면적으로 훑게 된다
  • 할루시네이션 — 검증 없이 생성된 숫자가 슬쩍 끼어든다

해결책은 전략컨설팅 펌이 수십 년 전에 이미 만들어뒀다. 분업과 교차검증.

8명 에이전트팀 아키텍처​

[PHASE 0]  오케스트레이터 ──→ MECE 분해, 핵심 질문 도출
              │
[PHASE 1]  ┌─┴──────────┬──────────────┐
           │            │              │
        리서처A        리서처B         리서처C
       시장/산업      기업/재무       기술/트렌드
        (웹검색)      (웹검색)        (웹검색)
           │            │              │
[PHASE 2]  └─────────┬──┘──────────────┘
                  ┌──┴──┐
            팩트체커A  팩트체커B
            숫자 검증  출처 검증
                  └──┬──┘
[PHASE 3]       구성작가
              피라미드 원칙으로
              마크다운 보고서 작성
                    │
[PHASE 4]       게이트키퍼
             로지컬씽킹 + MECE
                최종 승인

역할 8개가 5개 페이즈로 순서대로 돌아간다. 핵심은 단계별 검증이다. 리서처가 수집한 데이터를 팩트체커가 독립적으로 검증하고, 게이트키퍼가 최종 논리를 점검한다.

3가지 설계 원칙​

1. 한 명에게 다 시키지 않는다​

분업의 근거는 인지 부하다. 시장 분석과 재무 분석과 기술 분석을 동시에 수행하는 에이전트는 각 축의 깊이를 희생한다. 리서처 3명이 각자 전문 체크리스트를 가지고 병렬로 조사한다.

2. 자기들끼리 단계별 검증한다​

팩트체커가 2명인 이유가 있다. 숫자가 맞는지와 그 숫자의 출처가 신뢰할 수 있는지는 별개의 검증 작업이다. 예를 들어 "쿠팡 매출 35조"는 두 번 검증된다.

  • 팩트체커 A: 35조라는 숫자 자체가 정확한가
  • 팩트체커 B: 그 숫자를 뒷받침하는 출처가 공신력 있는가

3. 컨설팅 사고법으로 묶는다​

분업만으론 부족하다. 에이전트들이 같은 사고 프레임으로 작동해야 결과가 정돈된다.

프레임워크적용 단계역할
MECEPHASE 0, 4중복 없이, 빠짐 없이 — 3개 축이 전체 질문을 커버하는지 점검
로지컬 씽킹PHASE 2, 4숫자가 맞는지가 아니라 결론이 논리적으로 도출되는지
피라미드 원칙PHASE 3결론 먼저, 근거로 뒷받침 — 바쁜 독자를 위한 구조

Claude Code 스킬로 구현​

이 워크플로우를 재사용 가능한 스킬로 만들었다.

스킬 구조​

research-agent-team/
├── SKILL.md                      # 5단계 파이프라인 지시서
└── references/
    └── agent-personas.md         # 에이전트 8명의 페르소나와 체크리스트

SKILL.md는 파이프라인 흐름을 담고, agent-personas.md는 각 에이전트의 마인드셋과 체크리스트를 담는다. Progressive Disclosure 원칙 — 스킬이 트리거될 때만 SKILL.md가 로드되고, 페르소나 파일은 필요할 때 참조된다.

스킬 트리거 조건​

스킬 description에 명시된 트리거:

  • 기업/산업/시장/기술 리서치 요청
  • 비즈니스 인텔리전스 보고서 작성
  • 경쟁 분석, 통계 팩트체크
  • 컨설팅 스타일 구조화 보고서 요구

실전 검증 — AI 광통신 기술 리서치​

스킬을 만든 후 즉시 실전에 투입했다. 주제: AI 광통신 기술 리서치 및 투자 전략.

PHASE 0 — 오케스트레이터의 MECE 분해​

오케스트레이터가 5개 핵심 질문을 도출하고 3개 축으로 배분했다.

핵심 질문 5개:
1. AI 광통신 시장 규모·성장률은?
2. 핵심 기술 스택과 주요 플레이어는?
3. 투자 가능한 순수 광통신 종목·ETF는?
4. 최적 투자 기간과 진입 전략은?
5. 리스크 요인은?

MECE 분해:
├── 시장/산업 축: 1, 4 (부분)
├── 기업/재무 축: 3, 4 (부분)
└── 기술/트렌드 축: 2, 5 (부분)

PHASE 1 — 리서처 3명 병렬 조사 결과​

**리서처 A (시장/산업)**가 확인한 핵심 수치:

  • AI 데이터센터 광인터커넥트 시장: 2025년 3.75B→2033년3.75B → 2033년 3.75B→2033년18.36B
  • Lumentum 전망: 2025년 18BTAM→2030년18B TAM → 2030년 18BTAM→2030년90B (CAGR 40%)
  • 800G 데이터콤 광모듈: 2025년 출하량 60% 성장 예상

**리서처 B (기업/재무)**가 확인한 핵심 데이터:

  • Lumentum(LITE): 최근 분기 매출 665.5M(YoY+65665.5M (YoY +65%), 다음 분기 가이던스 665.5M(YoY+65805M
  • Coherent(COHR): NVIDIA $2B 투자, InP 생산량 2027년까지 4배 확대
  • Marvell(MRVL): FY2026 Q2 매출 $2.006B (YoY +58%, 사상 최고치)

**리서처 C (기술/트렌드)**가 확인한 로드맵:

  • 플러그어블 트랜시버 → 실리콘 포토닉스 → CPO(코패키지드 옵틱스) 전환
  • 스위치 속도: 2025년 800G → 2027년 1.6T → 2030년 3.2T
  • NVIDIA Quantum-X 스위치: 115.2 Tbps, 전력 3.5배 절감

팩트체커 B(출처 검증)의 신뢰도 분류:

PHASE 3 — 구성작가의 피라미드 원칙 적용​

보고서 첫 문장이 결론이다:

"AI 광통신은 GPU 클러스터 간 병목을 해소하는 필수 인프라로, 2025~2030년 약 40% CAGR로 성장해 TAM 90B에도달할전망이다.NVIDIA가2026년3월단독으로90B에 도달할 전망이다. NVIDIA가 2026년 3월 단독으로 90B에도달할전망이다.NVIDIA가2026년3월단독으로6B를 Lumentum·Coherent·Marvell에 투자하면서 구조적 업사이클이 공식 확인됐다."

결론을 먼저 쓰고, 나머지 섹션이 그 결론을 뒷받침하는 구조다.

PHASE 4 — 게이트키퍼 체크리스트​

로지컬 씽킹 체크:

  • 모든 결론에 검증된 출처 존재
  • 리스크 요인 별도 섹션으로 균형 확보 (업황 사이클, 밸류에이션 과열)
  • 섹션 간 상충 데이터 없음

MECE 체크:

  • 시장/재무/기술 3개 축 모두 커버
  • PHASE 0에서 도출한 핵심 질문 5개 모두 답변 완료
  • 섹션 간 중복 없음

게이트키퍼 승인 → 최종 보고서 출력.

핵심 인사이트​

1. 역할 분리가 깊이를 만든다 리서처 3명은 각자의 체크리스트만 채운다. 시장 리서처는 TAM·CAGR·경쟁구도에만 집중하고, 재무 리서처는 IR·공시에만 집중한다. 집중이 깊이를 만든다.

2. 팩트체커 분리가 신뢰를 만든다 숫자 검증과 출처 검증을 같은 에이전트가 하면 확증 편향이 생긴다. 분리된 두 팩트체커가 독립적으로 작동해야 진짜 교차검증이 된다.

3. 페르소나가 일관성을 만든다 각 에이전트에게 "McKinsey 어게이지먼트 매니저", "셀사이드 애널리스트", "Gartner 기술 분석가" 같은 구체적 페르소나를 주면 사고 프레임이 일관된다. 막연한 "전문가처럼 분석해줘"보다 훨씬 구체적인 결과가 나온다.

스킬 활용 방법​

Claude Code에서 이 스킬은 리서치 관련 요청이 들어오면 자동으로 트리거된다.

# 예시 입력
"삼성전자 HBM 사업 현황과 SK하이닉스와의 경쟁 구도를 리서치해줘"
"2026년 국내 클라우드 시장 전망 보고서 작성해줘"
"오픈AI vs 앤트로픽 기업 비교 분석해줘"

스킬이 트리거되면 오케스트레이터가 PHASE 0부터 자동으로 시작한다. 각 페이즈 완료 시 한국어로 진행 상황을 알려준다.

결론​

챗봇은 똑똑한 인턴이다. 에이전트팀은 전문가 조직이다. 차이는 분업과 교차검증이다.

이 스킬의 핵심 설계 결정은 세 가지다.

  1. 분업 — 한 에이전트에게 모든 역할을 주지 않는다
  2. 독립 검증 — 팩트체커는 리서처의 작업에 의존하지 않고 독립적으로 검증한다
  3. 사고 프레임 — MECE, 로지컬 씽킹, 피라미드 원칙을 에이전트 페르소나에 내재화한다

업무용 리서치에 시간을 쏟고 있다면, 또는 AI 결과를 다시 검증하느라 더 오래 걸리는 상황이라면, 8명 에이전트팀을 써보길 권한다.
