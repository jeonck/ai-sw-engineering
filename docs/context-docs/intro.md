# 문서와 맥락이 곧 생산성이다

## 패러다임 전환

예전엔 문서가 "있으면 좋은 것"이었습니다. AI 시대에는 **좋은 문서와 구조가 곧 AI 생산성의 기반**입니다.

Martin Fowler는 AI 코딩 환경에서 중요한 개념으로 **context engineering**을 제시합니다.

> "모델에게 무작정 많은 정보를 주는 것이 아니라, 잘 구조화된 코드, 규칙, 문서, 예시, 지침을 **적절하게** 제공해야 성능이 좋아진다."

## 문서가 AI 컨텍스트가 된다

소프트웨어 공학의 산출물들이 새로운 역할을 얻었습니다.

| 공학 산출물 | 전통적 역할 | AI 시대의 역할 |
|------------|------------|----------------|
| 설계 문서 | 팀 공유 자료 | AI의 아키텍처 이해 가이드 |
| ADR | 결정 기록 | AI가 일관된 결정을 내리는 근거 |
| 테스트 전략 | 품질 계획 | AI 코드 생성의 명세 |
| 코드 구조 | 파일 조직 | AI가 올바른 위치에 코드를 놓는 가이드 |
| README | 프로젝트 소개 | AI의 첫 번째 컨텍스트 |
| CLAUDE.md | (새로운 역할) | AI 에이전트를 위한 실행 지침 |

## 실천 전략

### CLAUDE.md / AGENTS.md 작성

AI 도구(Claude, GitHub Copilot, Cursor 등)에게 프로젝트 컨텍스트를 제공하는 파일입니다.

포함 내용:
- 프로젝트 구조 및 목적
- 코딩 관례 및 금지 패턴
- 아키텍처 원칙
- 테스트 작성 방법
- PR 작성 방법

### 구조화된 코드 작성

잘 구조화된 코드는 AI가 더 정확하게 이해합니다.

- 명확한 디렉터리 구조
- 일관된 파일 명명 규칙
- 레이어 분리 (도메인, 유스케이스, 인프라)
- 인터페이스와 구현 분리
