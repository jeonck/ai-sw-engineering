# AI는 공학을 대체하는 게 아니라 증폭한다

## 핵심 명제

DORA는 AI가 조직의 현재 상태를 비추는 **거울이자 증폭기**처럼 작동한다고 봅니다.

- 공학 기반이 **강한** 조직 → AI로 더 큰 효과
- 공학 기반이 **약한** 조직 → AI로 더 빠르게 혼란과 부채 축적

따라서 AI 시대에 소프트웨어 공학이 재조명되는 건, AI가 공학을 필요 없게 해서가 아니라 **공학의 부재를 더 빨리 드러내기 때문**입니다.

## 증폭기 비유

```
AI 없는 팀:  ──────────────→  결과물 (느리지만 제어 가능)
공학 기반 강한 팀 + AI:  ══════════════════→  결과물 (빠르고 품질 높음)
공학 기반 약한 팀 + AI:  ~~~~~~~~~~~~~~~~~~→  결과물 (빠르지만 혼란)
```

AI는 방향을 정해주지 않습니다. 이미 가고 있던 방향을 더 빠르게 갈 뿐입니다.

## 무엇이 "공학 기반"인가

다음 요소들이 갖춰진 조직이 AI로 더 큰 효과를 얻습니다.

- 설계 원칙 (아키텍처 가이드라인, 패턴 가이드)
- 팀 협업 방식 (코드 리뷰 문화, PR 규칙)
- 품질 체계 (테스트 커버리지 목표, 정적 분석)
- 문서화 습관 (ADR, 설계 문서, README)
- 테스트 자동화 (CI/CD, 회귀 테스트)

이 중 하나라도 취약하면, AI는 그 취약함을 **더 빠르게** 드러냅니다.
