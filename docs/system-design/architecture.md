# 아키텍처와 품질속성

## AI 시대의 아키텍처 설계

소프트웨어 아키텍처는 AI 시대에 더욱 중요해졌습니다. AI가 만든 코드가 시스템 전체에 통합되려면, **명확한 아키텍처 경계**가 있어야 합니다.

## 핵심 아키텍처 패턴

### Clean Architecture

```
외부 (UI, DB, API)
    ↓
어댑터 레이어
    ↓
유스케이스 레이어     ← AI가 가장 잘 구현하는 영역
    ↓
도메인 레이어        ← 인간 엔지니어가 설계해야 하는 영역
```

AI는 유스케이스 구현을 빠르게 도울 수 있지만, **도메인 모델과 비즈니스 규칙**은 여전히 인간이 설계해야 합니다.

### MSA (Microservices Architecture)

AI 시대의 MSA는 더 세심한 설계가 필요합니다.

- AI가 서비스 간 경계를 존중하지 않는 코드를 생성할 수 있음
- API 계약(OpenAPI, Protobuf)을 명확히 정의해 AI 생성 코드의 경계 침범 방지
- 서비스 간 데이터 흐름을 문서화해 AI 컨텍스트로 활용

## 품질속성 (Quality Attributes)

아키텍처 결정에서 품질속성은 트레이드오프의 중심입니다.

| 품질속성 | 정의 | AI 시대의 의미 |
|---------|------|---------------|
| **성능** | 응답시간, 처리량 | AI API 호출 비용과 지연 최적화 |
| **가용성** | 업타임 보장 | AI 모델 장애 시 fallback 전략 |
| **보안** | 인증, 권한, 데이터 보호 | 프롬프트 인젝션, 데이터 유출 방지 |
| **유지보수성** | 변경 용이성 | AI 생성 코드의 리팩터링 가능성 |
| **확장성** | 부하 증가 대응 | LLM API 호출량 급증 대응 |
| **관찰가능성** | 시스템 상태 파악 | AI 모델 응답 품질 모니터링 |

## ADR: 아키텍처 결정 기록

Architecture Decision Record(ADR)은 AI 시대에 더욱 중요합니다.

**왜 중요한가:**
- AI에게 프로젝트 컨텍스트를 제공하는 핵심 문서
- 과거 결정의 이유를 기록해 AI가 일관된 방향으로 코드 생성
- 새로운 팀원(인간 + AI 에이전트)이 빠르게 컨텍스트 파악 가능

**ADR 템플릿:**
```markdown
# ADR-001: [결정 제목]

## 상태
[제안됨 / 수락됨 / 대체됨]

## 맥락
[왜 이 결정이 필요했는가]

## 결정
[어떤 결정을 내렸는가]

## 결과
[이 결정의 긍정적/부정적 결과]
```
