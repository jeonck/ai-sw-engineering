# LLMOps와 AI 시스템 운영

## 전통 SDLC의 확장

전통적인 소프트웨어 생명 주기(SDLC)에 AI 모델의 생명 주기가 통합되면서 **MLOps/LLMOps**라는 새로운 공학적 영역이 확장되었습니다.

### 기존 SDLC
```
요구사항 → 설계 → 구현 → 테스트 → 배포 → 운영
```

### AI 통합 SDLC
```
요구사항 → 설계 → 구현 → 테스트 → 배포 → 운영
                              ↑
데이터 수집 → 모델 학습/파인튜닝 → 모델 평가 → 모델 배포 → 모델 모니터링
```

## LLMOps의 핵심 요소

### 1. 파이프라인 구축

데이터 수집, 학습, 배포, 모니터링에 이르는 전 과정을 **자동화하고 표준화**하는 공학적 접근이 필요합니다.

- 프롬프트 버전 관리
- 모델 평가 자동화
- A/B 테스트 인프라
- 모델 응답 로깅

### 2. 버전 관리의 확장

코드뿐만 아니라 다음도 버전 관리 대상이 됩니다.

- 모델 가중치 및 설정
- 학습 데이터셋
- 프롬프트 템플릿
- 평가 기준(evals)

### 3. 관측 가능성 (Observability)

LLM 기반 애플리케이션에서 관측 가능성은 더 복잡합니다.

| 지표 | 의미 |
|------|------|
| 레이턴시 | API 호출 응답 시간 |
| 토큰 사용량 | 비용과 직결 |
| 오류율 | API 실패, 타임아웃 |
| 품질 지표 | 응답의 정확성, 유용성 |
| 환각 감지율 | Hallucination 발생 빈도 |

## AI 시스템의 운영 현실

AI 시스템은 전통 소프트웨어와 다른 운영 패턴이 있습니다.

- **비결정론적 동작**: 같은 입력에 다른 출력이 나올 수 있음
- **외부 모델 의존성**: OpenAI, Anthropic 등 외부 API 의존
- **컨텍스트 관리**: 긴 대화 컨텍스트의 효율적 관리
- **프롬프트 드리프트**: 시간이 지남에 따라 같은 프롬프트의 결과 변화

이런 특성 때문에 **소프트웨어 공학적 운영 체계**가 더욱 중요합니다.
