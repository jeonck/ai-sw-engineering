лі‘л ¬ мІҳлҰ¬лҘј нҶөн•ң л¬ём ң к·ңлӘЁмқҳ нҷ•мһҘ к°ҖлҠҘм„ұ, Gustafsonмқҳ лІ•м№ҷ

I. к°ҖліҖ к·ңлӘЁмқҳ лі‘л ¬ мІҳлҰ¬ мӣҗлҰ¬, Gustafsonмқҳ лІ•м№ҷ к°ңмҡ”вҖӢ

м •мқҳ: лҢҖк·ңлӘЁ лҚ°мқҙн„° мІҳлҰ¬к°Җ н•„мҡ”н•ң л¬ём ңм—җм„ң лі‘л ¬ н”„лЎңм„ём„ң мҲҳлҘј лҠҳлҰ¬л©ҙ, мӢӨн–ү мӢңк°„мқ„ лҠҳлҰ¬м§Җ м•Ҡкі лҸ„ лҚ” нҒ° к·ңлӘЁмқҳ л¬ём ңлҘј мІҳлҰ¬н•  мҲҳ мһҲлӢӨлҠ” лі‘л ¬ м»ҙн“ЁнҢ…мқҳ лІ•м№ҷ

нҠ№м§•:
( Scaled Speedup ) л¬ём ңмқҳ нҒ¬кё°лҘј кі м •н•ҳм§Җ м•Ҡкі  мһҗмӣҗ мҰқк°Җм—җ л§һм¶° к·ңлӘЁлҘј нӮӨмҡё л•Ңмқҳ м„ұлҠҘ н–ҘмғҒ(нҷ•мһҘм„ұ)м—җ мЈјлӘ©н•Ё
( мҲңм°Ё мӢӨн–үмқҳ нқ¬м„қ ) лҚ°мқҙн„° к·ңлӘЁк°Җ м»Өм§ҲмҲҳлЎқ м „мІҙ мӢӨн–ү мӢңк°„м—җм„ң мҲңм°Ё мІҳлҰ¬ л¶Җл¶„(Serial Part)мқҙ м°Ём§Җн•ҳлҠ” л№„мӨ‘мқҙ мғҒлҢҖм ҒмңјлЎң мһ‘м•„м§җ
( кёҚм •м Ғ кҙҖм җ ) лі‘л ¬нҷ”к°Җ л¶Ҳк°ҖлҠҘн•ң л¶Җл¶„мқҙ м„ұлҠҘмқҳ м ҲлҢҖм Ғ н•ңкі„лҘј кІ°м •н•ңлӢӨлҠ” м•”лӢ¬(Amdahl)мқҳ лІ•м№ҷліҙлӢӨ лӮҷкҙҖм Ғмқҙкі  мӢӨл¬ҙм Ғмқё мӢңк°Ғмқ„ м ңкіөн•Ё

II. Gustafsonмқҳ лІ•м№ҷмқҳ л©”м»ӨлӢҲмҰҳкіј нҳ•мғҒнҷ”вҖӢ

к°Җ. лҚ°мқҙн„° к·ңлӘЁмҷҖ лі‘л ¬ мІҳлҰ¬мқҳ кө¬мЎ°м Ғ нҷ•мһҘ лӘЁлҚёвҖӢ

лӮҳ. Gustafsonмқҳ лІ•м№ҷ vs Amdahlмқҳ лІ•м№ҷ л№„көҗвҖӢ

л№„көҗ н•ӯлӘ©Amdahlмқҳ лІ•м№ҷ (кі м • к·ңлӘЁ)Gustafsonмқҳ лІ•м№ҷ (к°ҖліҖ к·ңлӘЁ)
кҙҖм җн•ҳл“ңмӣЁм–ҙ м„ұлҠҘмқҳ м ҲлҢҖм Ғ н•ңкі„ к°•мЎ°н•ҳл“ңмӣЁм–ҙ нҷ•мһҘмқ„ нҶөн•ң л¬ём ң н•ҙкІ° лҠҘл Ҙ к°•мЎ°
н•өмӢ¬ м „м ңл¬ём ңмқҳ нҒ¬кё°(Problem Size)к°Җ кі м •лҗЁмӢӨн–ү мӢңк°„(Execution Time)мқҙ кі м •лҗЁ
мҶҚлҸ„ н–ҘмғҒмҲңм°Ё мӢӨн–ү л¶Җл¶„м—җ мқҳн•ҙ мғҒн•ңм„ мқҙ кІ°м •лҗЁн”„лЎңм„ём„ң мҲҳм—җ л”°лқј м„ нҳ•м ҒмңјлЎң мҰқк°Җ к°ҖлҠҘ
мЈјмҡ” нҷңмҡ©м•Ңкі лҰ¬мҰҳмқҳ лі‘л ¬нҷ” нҡЁмңЁм„ұ нҸүк°Җл№…лҚ°мқҙн„°, мҲҳнҚјм»ҙн“ЁнҢ…, нҒҙлқјмҡ°л“ң нҷ•мһҘм„ұ м„Өкі„

III. мҶҢн”„нҠёмӣЁм–ҙ м„Өкі„м—җм„ңмқҳ Gustafsonмқҳ лІ•м№ҷ нҷңмҡ© м „лһөвҖӢ

к°Җ. мӢңмҠӨн…ң нҷ•мһҘм„ұ(Scalability) мөңм Ғнҷ” л°©м•ҲвҖӢ

м „лһөмғҒм„ё лӮҙмҡ©кё°лҢҖ нҡЁкіј
Horizontal Scalingкі м„ұлҠҘ мһҘл№„ көҗмІҙліҙлӢӨ м ҖмӮ¬м–‘ мһҘл№„мқҳ лҢҖлҹү лі‘л ¬нҷ”л№„мҡ© нҡЁмңЁм Ғмқё лҢҖк·ңлӘЁ мӣҢнҒ¬лЎңл“ң мІҳлҰ¬
Data PartitioningлҚ°мқҙн„°лҘј лҸ…лҰҪм Ғмқё лӢЁмң„лЎң мӘјк°ңм–ҙ л¶„мӮ° мІҳлҰ¬мһҗмӣҗ м¶”к°Җ мӢң мІҳлҰ¬ к°ҖлҠҘн•ң лҚ°мқҙн„° мҡ©лҹү мҰүк°Ғ нҷ•лҢҖ
Distributed ComputingSpark, Flink л“ұ л¶„мӮ° мІҳлҰ¬ н”„л Ҳмһ„мӣҢнҒ¬ нҷңмҡ©м•”лӢ¬мқҳ н•ңкі„лҘј л„ҳм–ҙм„  н…Ңлқјл°”мқҙнҠёкёү лҚ°мқҙн„° л¶„м„қ

лӮҳ. к°ңл°ң мӢң мӢңмӮ¬м җвҖӢ

  • Scaling for Insight: лӢЁмҲңнһҲ л№ лҘҙкІҢ мӢӨн–үн•ҳлҠ” кІғліҙлӢӨ, лҸҷмқјн•ң мӢңк°„ лӮҙм—җ лҚ” м •л°Җн•ҳкі  л°©лҢҖн•ң лҚ°мқҙн„°лҘј л¶„м„қн•ҳм—¬ мқёмӮ¬мқҙнҠёмқҳ м§Ҳмқ„ лҶ’мқҙлҠ” лҚ° мҙҲм җмқ„ л§һм¶°м•ј н•Ё
  • Cloud Native Design: нҒҙлқјмҡ°л“ң нҷҳкІҪм—җм„ңлҠ” мһҗмӣҗ(Node) м¶”к°Җк°Җ мҡ©мқҙн•ҳлҜҖлЎң, Gustafsonмқҳ лІ•м№ҷмқ„ м Ғмҡ©н•ҳм—¬ л№„мҰҲлӢҲмҠӨ м„ұмһҘм—җ л”°лҘё мӢңмҠӨн…ң нҷ•мһҘ кө¬мЎ°лҘј лҜёлҰ¬ м„Өкі„н•ҙм•ј н•Ё
