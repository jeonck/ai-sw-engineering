# Supervised Software Engineering Agents

## 개념: 위임과 통제의 균형

Thoughtworks가 제시하는 **supervised software engineering agents**는 개발자가 통제권을 유지한 채 더 큰 작업 범위를 AI에게 위임하는 흐름입니다.

핵심 질문:
- **어떤 범위**를 AI에게 맡길 것인가?
- **무엇을 자동화**하고 **무엇을 인간 판단**으로 남길 것인가?

## 위임 가능한 작업의 스펙트럼

```
완전 자동화 가능          인간 검토 필요          인간이 결정
        ↓                       ↓                    ↓
코드 포매팅           비즈니스 로직 구현        아키텍처 결정
린팅 수정             테스트 작성              보안 정책
보일러플레이트         문서화                  기술 스택 선택
CRUD 생성             코드 리뷰 보조           서비스 경계 설계
```

## AI 에이전트 감독 원칙

### 1. 명확한 작업 범위 설정

AI 에이전트에게 작업을 맡길 때 범위를 명확히 합니다.

```
좋은 예시:
"src/usecases/CreateOrder.ts 파일을 작성해줘.
 OrderRepository 인터페이스를 사용하고,
 domain/Order.ts의 Order 클래스를 생성해.
 다른 파일은 수정하지 마."

나쁜 예시:
"주문 기능을 만들어줘."
```

### 2. 체크포인트 설정

긴 작업을 여러 단계로 나누고 각 단계에서 검토합니다.

```
단계 1: 도메인 모델 설계 → 검토
단계 2: 인터페이스 정의 → 검토
단계 3: 유스케이스 구현 → 검토
단계 4: 테스트 작성 → 검토
단계 5: 통합 및 최종 검토
```

### 3. 롤백 가능한 단위 유지

AI가 만든 변경사항을 쉽게 롤백할 수 있도록 작은 커밋 단위를 유지합니다.

### 4. 사후 검증 루프

AI 에이전트가 작업한 결과를 자동화 테스트로 즉시 검증하는 루프를 구축합니다.

## 개발자의 AI 운영 능력

AI 시대의 개발자는 **AI 에이전트를 운영하는 능력**을 갖춰야 합니다.

- AI 결과물의 품질을 빠르게 평가하는 능력
- 좋은 프롬프트/컨텍스트를 설계하는 능력
- AI 실패 패턴을 인식하고 대응하는 능력
- AI와 사람의 협업 흐름을 설계하는 능력

이것은 단순한 "AI 사용법"이 아니라, **소프트웨어 공학적 판단력**을 AI 워크플로우에 적용하는 역량입니다.
